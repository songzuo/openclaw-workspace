# 团队沟通机制

## 定期会议系统

### 1. 每日站会 (Daily Standup)
- **时间**: 每天早上9点
- **参与者**: 所有子代理 + 主人
- **内容**: 昨天完成的工作、今天计划、遇到的问题
- **触发方式**: cron定时任务

### 2. 周规划会 (Weekly Planning)
- **时间**: 每周一上午
- **参与者**: 架构师 + 执行官 + 主人
- **内容**: 本周目标、任务分配、优先级

### 3. 问题研讨会 (Problem Solving)
- **触发**: 遇到复杂问题时由母代理发起
- **参与者**: 相关子代理 + 主人
- **内容**: 分析问题、讨论方案、决策

## 沟通渠道

### 方式1: 消息转发
```bash
# 母代理可以给指定子代理发消息
sessions_send sessionKey:"agent:main:subagent:xxx" message:"讨论一下这个架构方案"

# 子代理之间也可以沟通
sessions_send sessionKey:"agent:main:subagent:architect" message:"需要你评审一下代码"
```

### 方式2: 广播通知
```bash
# 重要通知发送给所有人
# 可以通过cron的wake功能触发会议
```

### 方式3: 共享上下文
- 所有子代理可以访问共享的记忆文件
- 重要决策记录到 `memory/meeting-notes.md`

## 会议记录

会议内容会记录到:
```
memory/meetings/
├── 2026-03-05-daily.md    # 每日站会
├── 2026-03-03-weekly.md   # 周规划
└── 2026-03-04-problem.md  # 问题研讨
```

## 发起临时会议

你可以说:
- "开个小会讨论一下这个问题"
- "请所有子代理汇报一下进度"
- "来个工作坊分析一下架构"
